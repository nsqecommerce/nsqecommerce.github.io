1. Web Calculator

Objective

สร้างโปรแกรม Web Calculator (เครื่องคิดเลขบนเว็บไซต์) โดยใช้ React Framework และ ภาษา Javascript หรือ TypeScript ในการพัฒนา

Requirements

  • ผู้ใช้งานสามารถ กดปุ่มตัวเลขบนหน้าจอตัวอย่าง และ มีตัวเลขขึ้นในช่องแสดงผล เหมือนเครื่องคิดเลขในชีวิตประจำวันของเรา
  • สามารถ บวก ลบ คูณ หาร ตัวเลข 2 ค่า ตามรูปตัวอย่างได้. และ เมื่อกด = แสดงผลลัพธ์ได้อย่างถูกต้อง
  • มีปุ่ม AC สำหรับ ล้างค่าที่กด เพื่อเริ่มต้นใช้งานใหม่ได้อีกครั้ง
  • จัดเรียงปุ่มให้มีลักษณะเหมือนเครื่องคิดเลขจริงๆ (ไม่ต้องมีสีสันสวยงามมาก)

Bonus (ไม่จำเป็นต้องทำก็ได้ หรือ เลือกทำบางข้อได้)

  • จัดเรียงหน้าตาให้สวยงาม
  • เปลี่ยนสี เปลี่ยน ธีม ของเครื่องคิดเลขได้
  • Support Responsive หน้าจอต่างๆ ได้

Evaluation Criteria

  • React best practices
  • อ่าน Code เข้าใจง่าย และ ง่ายต่อการนำไปต่อยอดได้
  • โปรแกรมสามารถทำงานได้ ตรงตาม Requirements
  • มีการจัดการข้อผิดพลาดจากการใช้งานโปรแกรม (Error handling)

2. Expense Tracker

Objective

สร้างโปรแกรม Expense Tracker (โปรแกรมบันทึกรายรับรายจ่าย) อย่างง่าย โดยใช้ React Framework และ ภาษา Javascript หรือ TypeScript ในการพัฒนา

Requirements

  • ผู้ใช้งานสามารถ (สร้าง/แก้ไข/ลบ) รายการ รายรับ รายจ่าย ได้ โดยมีข้อมูลดังนี้
    • วันที่ และ เวลา ของ รายการ
    • ชื่อ
    • ประเภทรายการ (รายรับ / รายจ่าย)
    • หมวดหมู่ ของรายการ
    • จำนวนเงิน
  • แสดงตารางรายการ ที่บันทึก
  • จัดเรียง และ กรอง ข้อมูลตาม วันที่, ชื่อ, หมวดหมู่, จำนวนเงิน และ ประเภทรายการ ได้
  • แสดง Chart สำหรับสรุปผลของ รายรับ และ รายจ่าย ตามหมวดหมู่ และ ตามระยะเวลาที่กำหนดได้ (ตามดุลพินิจของผู้พัฒนา)
  • ผู้ใช้สามารถ เพิ่ม/ลบ/แก้ไข หมวดหมู่ได้
  • ให้ logic การคำนวณทั้งหมดอยู่ที่โปรแกรม Front end
  • Design UI ของโปรแกรม ให้ใช้งานง่าย (ตามดุลพินิจของผู้พัฒนา)
  • โปรแกรมรองรับการใช้งานได้หลากหลายหน้าจอ (responsive)
  • ไม่จำเป็นต้องมีระบบผู้ใช้งาน

Bonus (เลือกทำบางข้อได้)

  • พัฒนา Front-End Framework โดยใช้ Ant Design
  • พัฒนาโปรแกรม Backend สำหรับบันทึกข้อมูล โดยรับข้อมูลผ่าน API (เลือกใช้ Stack ที่ถนัดได้เลย)
  • บันทึกข้อมูลในโปรแกรม Browser ป้องกันข้อมูลสูญหายเมื่อ Refresh Browser
  • สร้าง docker-compose.yml ให้ผู้ที่ checkout code ใช้งานได้ทันที

Evaluation Criteria

  • React best practices
  • อ่าน Code เข้าใจง่าย และ ง่ายต่อการนำไปต่อยอดได้
  • โปรแกรมสามารถทำงานได้ ตรงตาม Requirements
  • มีการจัดการข้อผิดพลาดจากการใช้งานโปรแกรม (Error handling)
